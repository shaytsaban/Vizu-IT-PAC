ארכיטקטורת מערכות מידע פיננסיות

מדריך אינטראקטיבי לבניית מערכת מורכבת: משיכת מספר מניות במקביל, סידור בסדרות-זמן, בקרות קלט צד-לקוח, וזרימת עבודה חכמה (טען -> צפה -> ייצא).

🚀

ברוכים הבאים לאתגר!

במטלה זו אתם הולכים לבנות כלי עבודה עוצמתי ברמת מוצר אמיתי, עם עיצוב פרימיום מרהיב.

המערכת שתבנו תאפשר למשתמש להקליד מניות רבות יחד, ותבצע בקרות (ולידציות) בזמן אמת כדי לוודא שלא הוקלדו תאריכים שגויים. הנתונים יוצגו קודם כל כתצוגה מקדימה במבנה של סדרת זמן (Time-Series) - כל שורה היא תאריך, וכל מניה מקבלת עמודה משלה להשוואה קלה.

🛡️

יצירת השרת: אלגוריתם סדרות זמן

השרת שלנו מורכב. הוא צריך לאסוף שערי סגירה מתואמים למספר מניות שונות, ולסנכרן אותם לאותה שורת תאריך בדיוק, כך שנקבל טבלה השוואתית מושלמת במבנה רחב.

📝 משימות לביצוע:

  1. היכנסו ל-script.google.com ולחצו + פרויקט חדש.
  2. שנו את שם הפרויקט למעלה ל-"Advanced Stock API".
  3. בעורך הקוד מחקו הכל.
  4. העתיקו את הפרומפט הבא ל-AI והדביקו את התוצר בעורך של גוגל.
ארכיטקטורת Pivot בשרת: השרת בונה "מפת תאריכים" (Date Map). הוא עובר מניה-מניה. עבור אפל, הוא ממלא את המחיר לכל תאריך. עבור מיקרוסופט, הוא מחפש את אותו תאריך בדיוק ומוסיף את המחיר לצידו. רק בסוף, הוא משטח את המפה לטבלה דו-ממדית (מערך של מערכים) ושולח אותה חזרה. בנוסף, יש כאן לוגיקת נפילה (Fallback) חכמה - אם ה-API הרגיל נכשל, הוא יוצר גיליון זמני באלפית שניה, שואב ממנו את נתוני Google Finance, ומוחק אותו מבלי שהמשתמש ירגיש.
🤖 הפרומפט להקמת השרת (Backend):

Act as an expert Google Apps Script developer. **CRITICAL FIRST STEP - SELF CHECKLIST:** Research: "Google Apps Script doPost CORS missing parameters e.postData.contents". Create a CHECKLIST explaining how you prevent payload parsing failures. Output this FIRST. Write a robust standalone script with `doGet(e)` and `doPost(e)`. **CRITICAL Payload Parsing:** The frontend sends `text/plain`. You MUST parse it safely: ```javascript let params = {}; try { const raw = e.postData.contents || (e.postData.getDataAsString ? e.postData.getDataAsString() : "{}"); params = JSON.parse(raw); } catch(err) { return ContentService.createTextOutput(JSON.stringify({status: "error", message: "Parse failed"})).setMimeType(ContentService.MimeType.JSON); } // MUST EXACTLY BE: tickers, startDate, endDate, frequency, action const { tickers, startDate, endDate, frequency, action } = params; if (!tickers || !startDate || !endDate || !frequency || !action) { throw new Error("Missing parameters. Received: " + JSON.stringify(params)); } ``` **ALGORITHM: TIME-SERIES WIDE FORMAT:** We want a pivot-style table. Column 1: "Date" (תאריך). Columns 2+: The specific ticker symbols. - Split `tickers` by comma, uppercase, and trim. - Create a `dateMap = {}` object to store data by date. - Loop through each ticker. - **Primary Source (Yahoo Finance):** Fetch CSV. Find "Date" column and "Adj Close" (or "Close") column. For each row, populate the map: `dateMap[dateObj][ticker] = closePrice`. - **Fallback (Google Finance):** If Yahoo fails, use a temporary sheet `=GOOGLEFINANCE("ticker", "price", "startDate", "endDate", "frequency")`. Extract Date and Close. Populate `dateMap[dateObj][ticker] = closePrice`. Delete temp sheet immediately! - Collect all unique dates from `dateMap`, sort them chronologically (oldest to newest). - Build `combinedData` 2D array. - Header: `["תאריך", ...tickersArray]` - Rows: Loop through sorted dates. First cell is the Date. Subsequent cells are the price for each ticker on that date (or "" if missing). **Action Handling:** - `action === 'csv'`: Convert `combinedData` to CSV string. Return `{"status": "success", "csvData": csvString}`. - `action === 'sheet'`: Create new Google Sheet. Write `combinedData`. Return `{"status": "success", "sheetUrl": url}`. - Catch overall errors and return `{"status": "error", "message": error.toString()}`. - Output ONLY the code with Hebrew comments.

פריסה (Deployment) והשגת ה-URL

🔑 תהליך הפריסה הקריטי:

  1. לחצו על הכפתור הכחול למעלה בעורך: פריסה (Deploy) -> פריסה חדשה (New deployment).
  2. בסוג (Type) בחרו אפליקציית אינטרנט (Web app).
  3. ⚠️ הגדרות חובה למניעת חסימות אבטחה:
    • הפעלה כ- (Execute as): Me / אני.
    • למי יש גישה (Who has access): Anyone / כל אחד.
  4. לחצו Deploy. אשרו הרשאות במסכים הקופצים (Advanced -> Go to project -> Allow).
  5. העתיקו ושמרו את ה-URL שנוצר! (מסתיים ב-/exec)

🐱 1. ההכנה ב-GitHub:

  1. צרו New repository (מאגר חדש) ב-GitHub. קראו לו stock-dashboard.

  3. לחצו Add file ואז Create new file.

🌍 3. הדלקת האתר לאוויר (GitHub Pages)

  1. לאחר ששמרתם את הקובץ, עברו ב-GitHub ללשונית Settings למעלה.
  2. בתפריט השמאלי, בחרו ב-Pages.
  3. תחת הכותרת Branch, שנו את הערך ל-main ולחצו Save.
  4. חובה להמתין: חיזרו לדף הראשי של המאגר שלכם. תראו שם נקודה צהובה קטנה מסתובבת. רק כשהיא הופכת ל-V ירוק (כ-2 דקות), תוכלו לחזור ל-Settings -> Pages, וללחוץ על הקישור הרשמי לאתר החי שלכם!

משהו נשבר או נתקע?

אל פחד, ככה מתקנים:

  1. מה הבעיה? לחצו על F12 באתר שלכם, כנסו ללשונית Console, והעתיקו את השגיאה האדומה ל-AI בבקשה לתיקון.
  2. תיקון בשרת? הדביקו קוד מתוקן ב-Apps Script -> חובה ללחוץ: New Deployment -> העתיקו את ה-URL החדש ושתלו אותו ב-HTML.
  3. תיקון ב-HTML? עדכנו את קוד ה-HTML ב-GitHub -> לחצו Commit -> המתינו ל-V הירוק -> רעננו את האתר עם Ctrl + F5.
