בניית מערכת לחילוץ נתוני שוק ההון

מדריך לסטודנטים: מ-HTML, דרך Google Sheets, ועד ל-Tableau – בעזרת בינה מלאכותית.

איך לומדים עם המדריך הזה?

במטלה זו אנו לא מצפים מכם להיות מתכנתים מדופלמים, אלא ארכיטקטי מידע. תפקידכם להבין את "התמונה הגדולה", את המושגים ואת השלבים. כדי לכתוב את הקוד בפועל, אתם תשתמשו בכלי בינה מלאכותית (כמו ChatGPT, Claude או Gemini). המדריך יסביר לכם מה קורה בכל שלב, ויספק לכם את ה"פקודה" (Prompt) המדויקת שעליכם לבקש מה-AI כדי להתקדם.

1

הבנת הארכיטקטורה: מי מדבר עם מי?

מה אנחנו הולכים לבנות?

לפני שמתחילים ללחוץ על כפתורים, בואו נבין את המבנה. דמיינו שאתם יושבים במסעדה. אתם לא יכולים פשוט להיכנס למטבח ולקחת אוכל. יש סדר פעולות ברור. במערכת שלנו יש שלושה שחקנים מרכזיים:

💡 אנלוגיית המסעדה:

  • צד הלקוח (התפריט והמלצר): זהו דף ה-HTML שניצור. המשתמש פותח אותו בדפדפן, רואה טופס (התפריט), מקליד איזה נתונים הוא רוצה (למשל מניית אפל) ולוחץ על "שלח" (קורא למלצר).
  • צד השרת (המטבח והשף): זהו ה-Google Apps Script (הקוד שנמצא בתוך גוגל שיטס). הוא מקבל את ההזמנה מהמלצר, מבין מה צריך לעשות, ומתחיל להכין את המנה.
  • API (ספק חומרי הגלם): השף במטבח לא מגדל את העגבניות בעצמו. כשהוא צריך נתוני מניות, הוא פונה לספק חיצוני (כמו Yahoo Finance) דרך "צינור" שנקרא API, מקבל את הנתונים, מסדר אותם יפה, ומגיש אותם לתוך גיליון ה-Google Sheets שלכם.

למה זה חשוב?

כי אם יש שגיאה, אתם צריכים לדעת איפה לחפש אותה. אם הטופס לא יפה – הבעיה בלקוח (HTML). אם הנתונים לא נשמרים בטבלה – הבעיה בשרת (Google).

2

הכנת מסד הנתונים: Google Sheets והשף הפרטי שלנו

מה אנחנו עושים כאן?

אנחנו צריכים "ארון תיוק" שבו ישמרו כל הנתונים המספריים שנוריד מהאינטרנט. ארון התיוק הזה הוא קובץ Google Sheets רגיל לחלוטין. אבל, גיליון רגיל לא יודע לעשות פעולות בעצמו. בשביל זה אנחנו מוסיפים לו "רובוט" - קוד שנקרא Google Apps Script (GAS).

הרובוט הזה מכיל פונקציה מיוחדת שנקראת doPost. חשבו עליה כעל תיבת דואר ייעודית של הרובוט. ברגע שנוחתת בתיבה הזו מעטפה (בקשה מהאתר שלנו), הרובוט מתעורר, קורא את הפתק (איזו מניה ביקשנו), הולך להביא את הנתונים וכותב אותם שורה אחר שורה בגיליון.

מה צריך לעשות בפועל?

  1. היכנסו ל-Google Drive שלכם וצרו גיליון Google Sheets חדש (לחצו כאן לפתיחה מהירה של גיליון חדש). קראו לו בשם ברור (למשל "נתוני מניות").
  2. בתפריט העליון של הגיליון, לחצו על הרחבות (Extensions) ואז בחרו ב-Apps Script. ייפתח לכם חלון חדש שהוא עורך הקוד (כאן הרובוט שלנו גר).
  3. מחקו את כל מה שכתוב שם. העתיקו את הפקודה (הפרומפט) הבאה, הדביקו אותה בצ'אט של בינה מלאכותית (כמו ChatGPT), ואת הקוד שתקבלו משם – הדביקו בעורך של גוגל ולחצו על כפתור השמירה (סמל של דיסקט).
🤖 פרומפט להעתקה לבינה מלאכותית:

אני סטודנט שבונה מערכת נתונים ואין לי ידע מקדים בתכנות. כתוב לי קוד Google Apps Script המכיל פונקציית doPost(e). הפונקציה צריכה: 1. לקבל מה-request (מתוך e.postData.contents שמגיע כ-JSON) פרמטרים: ticker (סימול מניה), startDate, endDate. 2. להשתמש ב-UrlFetchApp כדי להוריד נתוני מניות היסטוריים מ-Yahoo Finance API בפורמט CSV (לפי התאריכים והטיקר שהתקבלו). 3. לפרסר את ה-CSV באמצעות Utilities.parseCsv(). 4. לנקות את הגיליון הפעיל (clearContents) ולכתוב את כל הנתונים החדשים (הכותרות והשורות) לתוך הגיליון הפעיל (Active Sheet). 5. להחזיר JSON response עם סטטוס 'success' או הודעת שגיאה ברורה אם משהו נכשל. אנא הוסף הערות מפורטות בעברית מעל כל שורת קוד, כדי שאבין בדיוק מה היא עושה.

3

פריסת הסקריפט: יצירת הכתובת הציבורית

למה צריך את השלב הזה?

כרגע, הקוד ששמתם ב-Apps Script נמצא בתוך החשבון הפרטי והמאובטח שלכם. אף אחד מבחוץ לא יכול לדבר איתו. כדי שדף האינטרנט שניצור בשלב הבא יוכל לשלוח לו נתונים, אנחנו חייבים לפתוח לו "דלת" לעולם החיצון. התהליך הזה נקרא פריסה (Deployment). בסופו, גוגל ייצר עבורנו כתובת אינטרנט ארוכה (URL). הכתובת הזו היא כמו מספר הטלפון הישיר של הרובוט שלנו.

הנחיות צעד-אחר-צעד:

  1. בעורך ה-Apps Script (איפה שהדבקתם את הקוד), חפשו למעלה כפתור כחול שכתוב עליו פריסה (Deploy).
  2. בתפריט שנפתח, בחרו ב-פריסה חדשה (New deployment).
  3. בחלון שיקפוץ, לחצו על סמל גלגל השיניים ⚙️ (Select type) ובחרו ב-אפליקציית אינטרנט (Web app).
  4. הגדרות קריטיות (אם לא תעשו זאת, זה לא יעבוד):
    • תחת Execute as (הפעלה כ-), ודאו שכתוב Me (אני).
    • תחת Who has access (למי יש גישה), שנו ל-Anyone (כל אחד). זה אומר שכל מי שיש לו את הכתובת יוכל להפעיל את הקוד ולמשוך נתונים לגיליון.
  5. לחצו על הכפתור Deploy (פריסה).
  6. שימו לב: גוגל עלול להקפיץ אזהרת אבטחה מפחידה ("Google hasn’t verified this app"). אל דאגה! זה הקוד שלכם. לחצו על Advanced (מתקדם) ואז על Go to script (unsafe) / המשך לסקריפט (לא בטוח) ואשרו הכל.
  7. בסיום, יופיע מסך עם Web app URL. העתיקו את הכתובת הזו ושמרו אותה בצד (בפנקס רשימות למשל). תזדקקו לה בשלב הבא!
4

בניית לוח הבקרה למשתמש (דף ה-HTML)

מה אנחנו הולכים לבנות?

עכשיו כשיש לנו מטבח מוכן (שרת) שמחכה להזמנות, אנחנו צריכים לבנות את "התפריט" ללקוח. אנו ניצור קובץ HTML אחד שמכיל הכל:

HTML: השלד. יוצר את הקופסאות שבהן המשתמש מקליד (שדות קלט).
CSS (Tailwind): הצבע. הופך את הדף מ-"מכוער כמו אתר משנות ה-90" לאתר מודרני ויפה.
JavaScript (JS): המנוע. קוד שרץ בדפדפן, לוקח את מה שהלקוח הקליד, אורז למעטפה, ושולח אותה לרובוט שלנו בגוגל באמצעות פקודה שנקראת fetch (שליח דואר).

איך עושים את זה?

  1. צרו תיקייה חדשה במחשב שלכם למטלה זו.
  2. בתוך התיקייה, צרו קובץ טקסט חדש ושנו את השם שלו ל-index.html (ודאו שהסיומת היא .html ולא .txt). פתחו אותו לעריכה. מומלץ להשתמש בעורך קוד מקצועי וחינמי כמו VS Code (לחצו להורדה), אבל אפשר גם עם פנקס רשימות פשוט (Notepad).
  3. העתיקו את הפרומפט הבא לבינה המלאכותית (ChatGPT וכו').
  4. רגע קריטי! בקוד שתקבלו בחזרה מה-AI, חפשו את המילה YOUR_WEB_APP_URL. מחקו אותה, והדביקו במקומה את הכתובת הארוכה (ה-URL) ששמרתם בשלב 3.
  5. שמרו את הקובץ. עכשיו פשוט לחצו עליו פעמיים והוא ייפתח בדפדפן שלכם. נסו אותו! הכניסו מניה (למשל TSLA עבור טסלה) ולחצו על הכפתור. אם הכל עבד – הנתונים יופיעו לכם ב-Google Sheets!
🤖 פרומפט להעתקה לבינה מלאכותית:

אני צריך קובץ HTML אחד בלבד שמהווה לוח בקרה פיננסי למשתמש. אין לי ידע בתכנות אז הכל חייב להיות בקובץ אחד. הקובץ צריך לכלול: 1. עיצוב נקי, מקצועי ומודרני באמצעות ספריית Tailwind CSS (דרך CDN). עיצוב שמתאים לעולמות של שוק ההון. 2. במרכז הדף, טופס הכולל 3 שדות קלט (Inputs) עם תוויות ברורות: - 'סימול מניה (Ticker)' (שדה טקסט, למשל AAPL). - 'תאריך התחלה' (שדה תאריך - date). - 'תאריך סיום' (שדה תאריך - date). - כפתור גדול ובולט: 'משוך נתונים לגיליון'. 3. קוד JavaScript שמונע את רענון הדף בעת לחיצה על הכפתור (e.preventDefault). 4. ה-JavaScript צריך לקחת את הערכים מהשדות, ולארוז אותם למבנה JSON. 5. הקוד ישלח את ה-JSON באמצעות הפקודה fetch() (בשיטת POST) לכתובת 'YOUR_WEB_APP_URL' (אני אחליף את הכתובת הזו בעצמי אחר כך). 6. חשוב מאוד: כאשר המשתמש לוחץ על הכפתור, הראה לו ספינר טעינה או טקסט "טוען נתונים, אנא המתן...", וכאשר הבקשה מסתיימת - הראה הודעת הצלחה (או שגיאה) ברורה. הוסף הערות הסבר בעברית בתוך קוד ה-JavaScript כדי שאבין מה כל קטע עושה.

5

הקסם מתחיל: ניתוח ויזואלי ב-Tableau

מה השגנו עד עכשיו ומה השלב הבא?

אם הגעתם לכאן, בנימו מערכת מידע שלמה! יש לכם דף (ממשק) ששולט ברובוט (שרת), שמוריד נתונים עצומים מהאינטרנט (API), ומסדר אותם יפה בטבלה. אבל נתונים גולמיים בטבלה הם משעממים וקשה להבין מהם משהו. האם המניה בעלייה? האם היו ירידות חדות? כאן נכנסת לתמונה תוכנת Tableau (טבלו). זוהי תוכנה לויזואליזציה של נתונים (BI). היא יודעת להתחבר לארון התיוק שלנו (Google Sheets) בזמן אמת, ולהפוך את השורות והעמודות לגרפים ודאשבורדים מדהימים.

איך מחברים את טבלו לנתונים?

  1. פתחו את תוכנת Tableau במחשב שלכם. אם אין לכם את התוכנה, לחצו כאן לקבלת רשיון סטודנט חינם לשנה (Tableau Desktop).
  2. ממש במסך הפתיחה, בצד שמאל יש תפריט כחול בשם Connect (התחבר). חפשו שם את הכותרת To a Server (לשרת).
  3. ברשימה שתפתח, לחצו על Google Drive.
  4. דפדפן האינטרנט שלכם ייפתח ויבקש מכם להתחבר לחשבון הגוגל שלכם. אשרו לטבלו גישה לגיליונות שלכם.
  5. לאחר האישור, חיזרו לטבלו. תופיע לכם רשימה של כל הקבצים שלכם בגוגל. בחרו את הגיליון שאליו הזרמתם את הנתונים בשלבים הקודמים.
  6. זהו! עכשיו תוכלו לגרור את שדה "התאריך" (Date) לציר ה-X, ואת שדה "שער הסגירה" (Close) לציר ה-Y, ולראות מיד את גרף המניה שלכם. כל פעם שתשתמשו באתר שבניתם כדי למשוך נתונים חדשים, פשוט לחצו "רענן" בטבלו, והגרף יתעדכן אוטומטית!
