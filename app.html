בניית מערכת לחילוץ נתוני שוק ההון

מדריך לסטודנטים: מ-HTML, דרך Google Sheets, ועד ל-Tableau – בעזרת בינה מלאכותית.

איך לומדים עם המדריך הזה?

במטלה זו אנו לא מצפים מכם להיות מתכנתים מדופלמים, אלא ארכיטקטי מידע. תפקידכם להבין את "התמונה הגדולה", את המושגים ואת השלבים. כדי לכתוב את הקוד בפועל, אתם תשתמשו בכלי בינה מלאכותית (כמו ChatGPT, Claude או Gemini). המדריך יסביר לכם מה קורה בכל שלב, ויספק לכם את ה"פקודה" (Prompt) המדויקת שעליכם לבקש מה-AI כדי להתקדם.

💡 זכרו: תכנות הוא 80% פתרון תקלות ו-20% כתיבת קוד. אם אתם נתקלים בשגיאה – הכל בסדר! זה חלק בלתי נפרד מתהליך הלמידה. למטה תמצאו מדריך לפתרון התקלות הנפוצות ביותר.

1

הבנת הארכיטקטורה: מי מדבר עם מי?

מה אנחנו הולכים לבנות?

לפני שמתחילים ללחוץ על כפתורים, בואו נבין את המבנה. דמיינו שאתם יושבים במסעדה. אתם לא יכולים פשוט להיכנס למטבח ולקחת אוכל. במערכת שלנו יש שלושה שחקנים מרכזיים:

  • צד הלקוח (התפריט והמלצר): זהו דף ה-HTML שניצור. המשתמש פותח אותו בדפדפן, מקליד איזה נתונים הוא רוצה ולוחץ "שלח".
  • צד השרת (המטבח והשף): ה-Google Apps Script שלנו. הוא מקבל את ההזמנה, מבין מה לעשות ומתחיל לעבוד.
  • API (ספק חומרי הגלם): השף מבקש נתוני מניות מספק חיצוני (כמו Yahoo Finance) דרך "צינור" API, ומגיש אותם לגיליון שלכם.
🤔 למה אנחנו מתחילים דווקא מ-Google Sheets? ▼
הערה חשובה: באופן כללי, אפשר לייצר סקריפט (GAS) לגמרי לבד, כפרויקט עצמאי ב-Google Drive, מבלי לפתוח גיליון לפני כן. אנו בוחרים ליצור את הסקריפט מתוך הגיליון כרגע רק לשם הנוחות – זה מאפשר לסקריפט להכיר את הגיליון באופן אוטומטי (מה שנקרא "Bound Script"). בהמשך הקורס נכיר גם טכניקות של סקריפטים עצמאיים!
2

הכנת מסד הנתונים והרובוט שלנו

מה צריך לעשות בפועל?

  1. היכנסו ל-Drive וצרו גיליון Google Sheets חדש (לחצו כאן לפתיחה מהירה). קראו לו "נתוני מניות".
  2. בתפריט העליון לחצו על הרחבות (Extensions) -> Apps Script. ייפתח עורך הקוד.
  3. מחקו את מה שכתוב שם, העתיקו את הפרומפט הבא ל-AI, ואת הקוד שתקבלו הדביקו ושמרו (💾).
🤔 מה בעצם ביקשנו מה-AI לכתוב? ▼
ביקשנו פונקציה בשם doPost. זו פונקציה מיוחדת של גוגל שתפקידה להיות "תיבת דואר" שמאזינה לבקשות מחוץ לגוגל. כשהיא מקבלת בקשה (למשל מניית טסלה מתאריך X ל-Y), היא משתמשת בפקודת רשת (UrlFetchApp) כדי לשאוב את המידע מאתר פיננסי, מסדרת את הטקסט, ומדפיסה אותו תא-אחר-תא בגיליון שלנו.
🤖 פרומפט להעתקה לבינה מלאכותית:

אני סטודנט שבונה מערכת נתונים ואין לי ידע מקדים בתכנות. כתוב לי קוד Google Apps Script המכיל פונקציית doPost(e). הפונקציה צריכה: 1. לקבל מה-request (מתוך e.postData.contents שמגיע כ-JSON) פרמטרים: ticker (סימול מניה), startDate, endDate. 2. להשתמש ב-UrlFetchApp כדי להוריד נתוני מניות היסטוריים מ-Yahoo Finance API בפורמט CSV (לפי התאריכים והטיקר שהתקבלו). 3. לפרסר את ה-CSV באמצעות Utilities.parseCsv(). 4. לנקות את הגיליון הפעיל (clearContents) ולכתוב את כל הנתונים החדשים (הכותרות והשורות) לתוך הגיליון הפעיל (Active Sheet). 5. להחזיר JSON response עם סטטוס 'success' או הודעת שגיאה ברורה אם משהו נכשל. אנא הוסף הערות מפורטות בעברית מעל כל שורת קוד, כדי שאבין בדיוק מה היא עושה.

3

פריסה (Deployment) ויצירת כתובת ציבורית

כדי שהאתר שניצור יוכל לתקשר עם הסקריפט בגוגל, אנחנו חייבים להפוך אותו לשירות אינטרנטי ציבורי ולקבל עבורו כתובת URL.

🚦 שלב קריטי מקדים: פריסת ניסיון (Test Deployment)

לפני שמשחררים קוד לציבור, מתכנתים תמיד בודקים אותו בסביבת "ארגז חול". זה מוודא שאין שגיאות קריטיות.

  1. לחצו על הכפתור הכחול למעלה פריסה (Deploy) -> בחרו פריסות ניסיון (Test deployments).
  2. לחצו על סמל גלגל השיניים ⚙️ ובחרו אפליקציית אינטרנט (Web app).
  3. גוגל יבקש מכם לאשר הרשאות (כי הסקריפט מנסה לגשת לגיליון ולרשת). אשרו הכל. (לחצו על Advanced -> Go to script).
  4. אם זה עבר בהצלחה וקיבלתם V ירוק, סימן שהקוד תקין ואפשר להמשיך לפריסה האמיתית!

יצירת הכתובת הסופית:

  1. שוב לחצו על פריסה (Deploy) -> והפעם פריסה חדשה (New deployment).
  2. הגדרות חובה:
    • הפעלה כ- (Execute as): Me (אני).
    • למי יש גישה (Who has access): Anyone (כל אחד). (בלי זה – התקשורת תיחסם!)
  3. לחצו על Deploy (פריסה).
  4. העתיקו את הכתובת שנוצרה (Web app URL) ושמרו אותה בפנקס רשימות!
4

בניית הדף ואירוח ב-GitHub Pages

עכשיו נבנה את "התפריט" ללקוח (HTML) ונעלה אותו כאתר חי באמצעות GitHub Pages.

💎 טיפ של אלופים (Prompt Optimization):

במקום לחפש שורות קוד ולהחליף משתנים ידנית (מקור להרבה טעויות), אפשר לשתול את הכתובת של הרובוט שהעתקתם בשלב הקודם ישירות בתוך הפרומפט! כך הבינה המלאכותית תייצר עבורכם קוד HTML מוכן שניתן להעתיק כפי שהוא מבלי לשנות בו אפילו פסיק.

🤷🏽‍♂️ אין לי גיטהאב בכלל! מה זה ואיך נרשמים? (לחצו למדריך בסיסי) ▼

גיטהאב (GitHub) היא הרשת החברתית וה"גוגל דרייב" של המתכנתים. כל הקוד הפתוח בעולם יושב שם. בשבילנו, זה שירות חינמי נהדר לאחסן את הקוד שלנו ולהציג אותו כאתר אינטרנט חי לכל דבר.

  1. היכנסו לאתר GitHub.com ולחצו על הכפתור Sign up (הרשמה) בצד ימין למעלה.
  2. הזינו כתובת אימייל, בחרו סיסמה חזקה, ובחרו שם משתמש (Username) באנגלית בלי רווחים. אשרו את כתובת המייל שלכם (תקבלו קוד אימות למייל).
  3. לאחר ההתחברות, תראו במסך הראשי כפתור ירוק New או סימן פלוס (+) למעלה בצד ימין -> ואז New repository. (ריפו/מאגר הוא פשוט התיקייה של הפרויקט שלכם).
  4. תחת Repository name, הקלידו שם קצר לאתר שלכם, למשל stock-dashboard.
  5. חשוב: ודאו שהמאגר מוגדר כ-Public (ציבורי), אחרת לא נוכל לארח אותו בחינם כאתר פתוח.
  6. גללו למטה ולחצו על הכפתור הירוק Create repository. זהו, יצרתם את הכספת שלכם! אפשר לחזור למדריך הראשי.

איך מפרסמים את האתר?

  1. פתחו משתמש ב-GitHub, וצרו מאגר (Repository) חדש כ-Public (היעזרו במדריך הסגול למעלה אם אתם מסתבכים).
  2. לחצו על Add file ואז Create new file. קראו לקובץ בשם המדויק index.html.
  3. העתיקו את הפרומפט למטה ל-AI (אל תשכחו להכניס אליו את ה-URL שלכם לפני השליחה!).
  4. הדביקו את קוד ה-HTML שקיבלתם לתוך גיטהאב ולחצו למעלה על הכפתור הירוק Commit changes (המשמעות: שמירת הגרסה בקוד).
🚀 הסבר מפורט: איך מדליקים את האתר ב-GitHub? (לחצו לפירוט) ▼

כברירת מחדל, גיטהאב משמש רק כ"כספת" לאחסון קוד. כדי להפוך את הכספת הזו לאתר חי שכולם יכולים לגלוש אליו, צריך להדליק את שירות ה-Pages.

  1. היכנסו ללשונית Settings (הגדרות) בסרגל העליון של המאגר שלכם.
  2. בתפריט השמאלי, גללו מטה ולחצו על Pages.
  3. תחת הכותרת Build and deployment, חפשו את Branch (ענף). שנו את הערך מ-None ל-main (זה אומר לגיטהאב: "קח את הקבצים מהענף הראשי שלי ותציג אותם כאתר").
  4. לחצו על Save.
  5. עכשיו צריך סבלנות: המתינו כ-2 דקות ורעננו את הדף. אתם תראו הודעה המכילה קישור ירוק/כחול שאומר "Your site is live at...". זה הלינק הרשמי לאתר שלכם!

כך נראה הקישור לאחר שהאתר באוויר.
🤖 פרומפט להעתקה לבינה מלאכותית:

אני צריך קובץ HTML אחד בלבד שמהווה לוח בקרה פיננסי למשתמש (אני הולך לארח אותו ב-GitHub Pages). הקובץ צריך לכלול: 1. עיצוב נקי, מקצועי ומודרני באמצעות Tailwind CSS. 2. טופס מרכזי עם 3 שדות קלט (Inputs): 'סימול מניה (Ticker)', 'תאריך התחלה', ו-'תאריך סיום'. בנוסף כפתור 'משוך נתונים לגיליון'. 3. קוד JavaScript שישדר את הנתונים מהטופס במבנה JSON באמצעות הפקודה fetch() בשיטת POST. 4. **קריטי:** כתובת ה-URL שאליה יש לשלוח את בקשת ה-fetch היא הכתובת הזו: [הדביקו לכאן את ה-URL הארוך שקיבלתם מגוגל בשלב 3 !!!] 5. הראה למשתמש ספינר טעינה בזמן ההמתנה, והודעת הצלחה/שגיאה בסיום. אנא ספק לי קוד HTML מוכן ושלם להעתקה, ללא צורך בעריכה נוספת מצדי.

5

הקסם מתחיל: ניתוח ויזואלי ב-Tableau

הגענו ליעד! הנתונים בגיליון. כעת נשתמש בתוכנת Tableau כדי להפוך שורות ועמודות משעממות לגרפים דינמיים.

  1. פתחו את תוכנת Tableau Desktop (קבלת רשיון סטודנט חינם לשנה).
  2. במסך הפתיחה, תחת To a Server, לחצו על Google Drive.
  3. אשרו את החיבור דרך הדפדפן (פעולה זו מאפשרת לטבלו לקרוא את המסמכים שלכם בזמן אמת).
  4. בחרו את הגיליון "נתוני מניות" שלכם. זהו! גררו את שדה התאריך לציר ה-X, ושער סגירה לציר ה-Y. מדהים, לא?

🛠️ נתקעתם? מדריך שגיאות נפוצות

אין מתכנת שלא חווה שגיאות. הנה הפתרונות לדברים הנפוצים ביותר שיכולים להשתבש:

❌ לחצתי "Run" בסקריפט של גוגל וקיבלתי שגיאה: Cannot read property 'postData'
הכל בסדר! כך זה אמור להיות. הפונקציה doPost(e) מצפה לקבל את האות 'e' (כלומר Event - נתונים שמגיעים מבחוץ, מהטופס שבאתר). כשאתם לוחצים Run בתוך העורך, אין שום נתונים שמגיעים, ולכן הסקריפט קורס. הבדיקה האמיתית היא דרך האתר שבניתם.
❌ שגיאת CORS מופיעה בקונסול של הדפדפן (F12)
זו השגיאה הנפוצה ביותר! היא אומרת שגוגל חוסם את הבקשה. זה קורה משתי סיבות:
  1. בשלב 3 שכחתם לשנות את Who has access ל-Anyone.
  2. עשיתם שינוי בקוד הסקריפט אבל לחצתם רק "שמור". חובה לעשות New Deployment בכל פעם שמשנים קוד בשרת! ואז לעדכן את ה-URL החדש בקוד ה-HTML שלכם.
❌ האתר אומר "הצלחה" אבל הגיליון נשאר ריק
ודאו שסימול המניה (Ticker) שהזנתם באמת קיים ב-Yahoo Finance. נסו להזין סימול מוכר כמו AAPL או MSFT. אם עדיין לא עובד, ייתכן שגוגל דורש לאשר מחדש את ההרשאות דרך פריסת הניסיון (Test Deployment - ראו שלב 3).
נוצר עבור קורס ניתוח נתונים ומערכות מידע • בהצלחה במטלה!
