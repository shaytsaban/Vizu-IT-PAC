בניית מערכת לחילוץ נתוני שוק ההון

מדריך פרקטי: מ-HTML, דרך Google Sheets, ועד לויזואליזציה ב-Tableau.

מטרת המטלה

במטלה זו אנו לא מצפים מכם להיות מתכנתים מדופלמים, אלא ארכיטקטי מידע. תפקידכם להבין את זרימת המידע, את בניית המערכת ואת הטיפול בתקלות. כדי לכתוב את הקוד בפועל, תשתמשו בכלי בינה מלאכותית (כמו ChatGPT, Claude או Gemini). המדריך מספק לכם את ה"פקודות" (Prompts) המדויקות – כולל מעקפי אבטחה מתקדמים – שעליכם לבקש מה-AI כדי לייצר מערכת חסינה.

1

הבנת הארכיטקטורה: סדר הפעולות

במערכת שלנו יש שלושה רכיבים מרכזיים שמדברים אחד עם השני. חשוב להבין מי עושה מה, כדי לדעת איפה לחפש שגיאות (Debugging):

  • צד הלקוח (Frontend): דף ה-HTML/JS. זהו ממשק המשתמש (הטופס). תפקידו לאסוף את שם המניה והתאריכים ולשלוח אותם הלאה. בעיות עיצוב או שגיאות שליחה (CORS) קורות כאן.
  • צד השרת (Backend): קוד ה-Google Apps Script (GAS) שיושב בתוך גוגל שיטס. הוא המוח שמקבל את הבקשה ומנהל את העבודה. בעיות של חוסר הרשאות מתרחשות פה.
  • ספק הנתונים (API): שרתי Yahoo Finance, שמהם נמשוך את המידע. יאהו חוסמים לעיתים רובוטים, ולכן נצטרך לכתוב לשרת שלנו "תוכנית גיבוי" שתמשוך נתונים מפונקציות גוגל מובניות במקרה של חסימה.
2

הכנת מסד הנתונים וקוד השרת (Google Apps Script)

ניצור קובץ Sheets שיהווה את מסד הנתונים שלנו, ונשתול בו קוד שיודע לתקשר עם Yahoo Finance או לעקוף אותו במקרה הצורך.

  1. פתחו גיליון Google Sheets חדש וקראו לו בשם ברור.
  2. בתפריט העליון, לחצו על הרחבות (Extensions) ואז על Apps Script.
  3. מחקו את כל הקוד הקיים שם.
  4. העתיקו את הפרומפט הבא ל-AI שלכם (הוא מכיל הנחיות קריטיות לעקיפת אבטחה), הדביקו את התשובה בעורך של גוגל, ושמרו.
🤖 פרומפט שרת - להעתקה לבינה מלאכותית:

אני צריך קוד Google Apps Script המכיל פונקציית doPost(e) שתשמש כ-Web App לחילוץ נתוני מניות. הקוד חייב לכלול את ההנחיות הקריטיות הבאות: 1. קבלת הנתונים מהבקשה מתוך e.postData.contents, וביצוע JSON.parse() כדי לחלץ את: ticker, startDate, endDate. 2. מנגנון משיכה מ-Yahoo Finance: לפני משיכת ה-CSV, על הקוד לשלוח בקשה ל-finance.yahoo.com כדי למשוך עוגייה (Cookie), ולאחר מכן לשלוח בקשה ל-getcrumb כדי לחלץ אסימון אבטחה. רק עם שניהם, יש לפנות ל-API של הורדת ה-CSV. 3. תוכנית גיבוי חובה: עטוף את כל תהליך יאהו ב-try-catch. אם יאהו חוסם את הבקשה (שגיאות 401/404/Crumb), בבלוק ה-catch, נקה את הגיליון, וכתוב לתא A1 נוסחת GOOGLEFINANCE שתמשוך את הנתונים ההיסטוריים (פתיחה, סגירה, גבוה, נמוך, נפח) לפי הטיקר והתאריכים שנשלחו. 4. בסוף, עליו להחזיר תגובת JSON (עם סטטוס success או error). הוסף הערות מפורטות בעברית מעל כל שורת קוד.

3

פריסת הסקריפט ויצירת כתובת URL

כדי שדף האינטרנט שלנו יוכל לשלוח פקודות לגיליון, חובה לחשוף את הקוד החוצה בצורה מאובטחת.

  1. לחצו על הכפתור הכחול למעלה: פריסה (Deploy).
  2. בחרו ב-פריסה חדשה (New deployment).
  3. לחצו על גלגל השיניים ⚙️ ובחרו אפליקציית אינטרנט (Web app).
  4. ⚠️ הגדרות קריטיות (השלב שבו רובם נופלים!):
    • תחת Execute as: חובה לבחור Me (אני).
    • תחת Who has access: חובה לשנות ל-Anyone (כל אחד).
  5. לחצו Deploy. אשרו את אזהרות האבטחה (Advanced -> Go to script (unsafe)).
  6. העתיקו את ה-Web app URL ושמרו אותו.
טיפ זהב: עשיתם שינוי כלשהו בקוד ב-Apps Script? הכתובת הישנה לא תתעדכן! אתם חייבים לעשות שוב "Deploy -> New deployment" כדי שהשינויים ייכנסו לתוקף. (אל תשתמשו ב-Manage deployments).
4

בניית לוח הבקרה למשתמש (HTML/JS)

כעת ניצור את "התפריט" - ממשק המשתמש שבו נקליד איזה נתונים אנו רוצים. אנו נשתמש בטריק מיוחד ב-JavaScript כדי למנוע חסימות דפדפן (CORS).

  1. צרו מאגר חדש (Public) ב-GitHub.
  2. צרו קובץ חדש בשם index.html.
  3. העתיקו את הפרומפט הבא ל-AI שלכם כדי לייצר את קוד ה-HTML.
  4. חשוב: בקוד שתקבלו, מצאו את המילה YOUR_WEB_APP_URL, והחליפו אותה בכתובת ששמרתם משלב 3.
  5. שמרו את הקובץ. הפעילו את GitHub Pages בהגדרות המאגר, והמתינו לקבלת הלינק לאתר החי שלכם.
🤖 פרומפט לקוח - להעתקה לבינה מלאכותית:

אני צריך קובץ HTML אחד המהווה לוח בקרה פיננסי. 1. עצב את הדף בצורה מודרנית עם Tailwind CSS. 2. במרכז, טופס עם: שדה 'סימול מניה' (טקסט), 'תאריך התחלה' (date), 'תאריך סיום' (date), וכפתור שליחה עם ספינר טעינה. 3. קוד JavaScript שמונע רענון דף ואוסף את הנתונים ל-JSON. 4. קריטי לעקיפת חסימות דפדפן (CORS Error): בקשת ה-fetch(URL) לשרת גוגל חייבת להיות בשיטת POST. בהגדרות ה-headers חובה להגדיר את ה-Content-Type להיות בדיוק: 'text/plain;charset=utf-8'. בשום פנים ואופן לא להשתמש ב-application/json. ה-body צריך להכיל את ה-JSON.stringify. הכתובת לבקשה תהיה 'YOUR_WEB_APP_URL'. 5. טיפול בתגובה: הצגת הודעת הצלחה ירוקה או שגיאה אדומה בהתאם לתגובה. הוסף הערות קוד בעברית.

5

ניתוח ויזואלי ב-Tableau

המערכת פועלת! הנתונים נמשכים אוטומטית לגיליון בלחיצת כפתור באתר שלכם. כעת ניקח את המספרים ונהפוך אותם לתובנות עסקיות באמצעות Tableau.

  1. פתחו את תוכנת Tableau Desktop במחשב.
  2. בתפריט השמאלי, תחת Connect -> To a Server, בחרו Google Drive.
  3. התחברו עם חשבון הגוגל שלכם ואשרו לטבלו לגשת לגיליונות.
  4. בחרו את גיליון נתוני המניות שיצרתם.
  5. עברו ל-Sheet1. גררו את שדה התאריך לציר ה-Columns, ואת שערי הסגירה או הנפח לציר ה-Rows כדי לייצר את הויזואליזציה שלכם.
  6. סנכרון חכם: בפעם הבאה שתשתמשו באתר שיצרתם כדי למשוך נתונים של מניה אחרת, הגיליון יתעדכן - וכל שעליכם לעשות הוא ללחוץ על "רענן" (Refresh) בטבלו כדי שהגרפים יתעדכנו מיידית!
