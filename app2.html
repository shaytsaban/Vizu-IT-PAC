אנחנו בונים את התשתית - הויזואליזציה בטבלו תגיע במפגש הבא!

מדוע ללמוד Scraping ואוטומציה?

במערכות פיננסיות, הנתונים הם "הדלק" של כל החלטה. כיום, רוב המידע נמצא בפורמטים לא נגישים (אתרי אינטרנט, קבצי PDF, APIs נעולים). סקרייפינג (Scraping) הוא הכוח להפוך את המידע הזה למסד נתונים (Dataset) מובנה.

זכרו: באגים הם חלק מהחיים. קוד שנשבר הוא בסך הכל חידה שמחכה לפתרון. אנחנו נלמד איך להשתמש ב-Console של הדפדפן ובכלי בינה מלאכותית כדי לנצח כל שגיאה.

📖 אנלוגיית המסעדה המורחבת:

הממשק (HTML): זהו ה"תפריט" והמלצר. המשתמש בוחר מנה (טיקר) ותוספות (תאריכים).
השרת (Google Script): זה ה"שף". הוא המתווך היחיד שיכול להיכנס למטבח, לחתוך את הנתונים ולסדר אותם בצלחת.
ה-API (המחסן): יאהו פייננס היא הספקית. הנתונים שם גולמיים ומלוכלכים, והשף שלנו דואג לנקות אותם עבורנו.

*הערה: את "הגשת המנה" (ויזואליזציה) נבצע בטבלו במפגש הבא.

שלב א': הקמת המוח הפיננסי (Backend)

אנחנו בונים שרת Standalone. הוא לא "עבד" של קובץ אקסל, אלא יחידת מחשוב עצמאית שמייצרת קבצים. זהו הלב של ה-Automation.

איך מתחילים?

  • היכנסו ל-script.google.com.
  • לחצו על New project.
  • שנו את השם ל-"Finance_Data_Engine".
עומק טכנולוגי: מדוע השרת הכרחי? Yahoo Finance חוסמת בוטים. הקוד שלנו מבצע "ריקוד אבטחה": הוא פונה קודם כל לקבלת Cookie (זהות) ואז ל-Crumb (אסימון אישור). בנוסף, השרת מבצע **Data Transformation**: הוא לוקח CSV גולמי (שהוא טקסט ארוך), מפרק אותו, ובורר מתוכו רק את עמודת התאריך וה-Adj Close. שאר הנתונים הם "רעש" לצרכי השוואת ביצועים.
🤖 פרומפט להעתקה לבינה מלאכותית (ChatGPT/Gemini):

אני בונה שרת ב-Google Apps Script. כתוב לי קוד הכולל פונקציית doGet(e) פשוטה שמחזירה "Server is running", ופונקציית doPost(e) שמבצעת: 1. קבלת JSON עם tickers (מערך), startDate, endDate ו-frequency. 2. מנגנון Cookie/Crumb מלא למשיכת CSV מ-Yahoo Finance לכל טיקר. 3. איחוד (Merge): חילוץ עמודות Date ו-Adj Close בלבד, ואיחוד כל המניות לטבלה אחת גדולה (Data Merging). 4. שימוש ב-SpreadsheetApp.create ליצירת גיליון גוגל חדש, כתיבת הטבלה אליו, והגדרת הרשאות Anyone with link. 5. החזרת ה-URL של הגיליון בתגובה כ-JSON (sheetUrl). הוסף הערות הסבר מפורטות בעברית לכל שלב בקוד.

שלב ב': פריסה (Deployment) וטסטים

🛑 חשיבות האבטחה (אל תבהלו!):

בזמן האישור, גוגל תזהיר אתכם שהאפליקציה אינה מאומתת. זכרו: זה הקוד שלכם! לחצו על Advanced -> Go to Project (unsafe). זהו שלב הכרחי בכל פיתוח של Web App פנימית.

ה-Deployment Test (כאן בודקים שאין באגים בשרת):

לאחר שתקבלו את ה-Web App URL, העתיקו אותו ופתחו אותו בדפדפן.

✓

תרחיש הצלחה:

מופיע הכיתוב "Server is running". המשמעות: ה-doGet עובד, והשרת מוכן לקבל פקודות.

!

תרחיש כישלון:

שגיאת 401/404 או בקשת התחברות. המשמעות: שכחתם להגדיר "Anyone" בפריסה. ה-HTML שלכם לא יצליח לדבר עם השרת.

שלב ג': GitHub Pages - האתר שלכם באוויר

🏗️ מדריך GitHub מפורט (צעד-אחר-צעד)

1. ה-Repository

היכנסו ל-GitHub וצרו "מאגר" חדש. תנו לו שם כמו `my-finance-portal`. ודאו שהוא מוגדר כ-Public. מאגר ציבורי הוא תנאי הכרחי לשימוש ב-GitHub Pages בגרסה החינמית.

2. הקובץ index.html

בתוך המאגר, לחצו על `Add file` -> `Create new file`. קראו לו בדיוק index.html. השם הזה קריטי כי השרת של גיטהאב מחפש אותו כנקודת הכניסה לאתר.

3. החלפת ה-URL (חשוב!)

בתוך הקוד שקיבלתם מה-AI, אתרו את המחרוזת `YOUR_WEB_APP_URL`. מחקו אותה והדביקו את הלינק שקיבלתם מגוגל בשלב הקודם (זה שעבר את הטסט).

4. הפעלת ה-Hosting

עברו ללשונית Settings -> Pages. תחת `Branch` בחרו ב-main ולחצו Save. המתינו דקה, רעננו, והלינק לאתר החי שלכם יופיע בראש הדף!

🤖 פרומפט לממשק המשתמש (The Web App):

כתוב לי קוד HTML/JS מלא בקובץ אחד המיועד לאחסון ב-GitHub Pages. עיצוב: מודרני, נקי, מבוסס Tailwind CSS, עם צבעים מעולמות הפיננסים (כחול כהה, אפור סלייט). פונקציונליות: 1. טופס המקבל רשימת סימולי מניות (מופרדים בפסיק, למשל TSLA, AAPL, AMZN). 2. שדות תאריכים ותפריט בחירת תדירות (Daily/Weekly/Monthly). 3. שליחה ב-fetch בשיטת POST לכתובת 'YOUR_WEB_APP_URL'. 4. הגדרת Headers כ-'text/plain;charset=utf-8' (למניעת שגיאות CORS). 5. לוגיקת טעינה: הראה "טוען נתונים..." על הכפתור בזמן השליחה. 6. לאחר ההצלחה: הצגת כפתור גדול שפותח את הלינק שחוזר מהשרת (sheetUrl) בטאב חדש. הוסף הערות הסבר בעברית.

🛠️ כלי עזר לפתרון באגים:

אם לחצתם על הכפתור ושום דבר לא קרה: לחצו על F12 במקלדת, עברו ללשונית Network ונסו שוב. שם תוכלו לראות את הבקשה יוצאת לגוגל ואם היא נחסמה (שגיאה אדומה). זהו הכלי הכי חשוב של מפתח מערכות מידע!

העתיד: ויזואליזציה והפקת תובנות

מזל טוב! בניתם מערכת ETL מלאה.

יצרתם "מפעל נתונים". האפליקציה שלכם יודעת לחלץ (Extract), לעבד (Transform) ולאחסן (Load) נתונים פיננסיים בצורה אוטונומית לחלוטין.

למה השקענו כל כך הרבה בצינור? כי הנתונים שיצרתם כרגע בגיליון הם נתונים "נקיים". הם מסוננים, מאוחדים ומיושרים לפי תאריכים. זהו המצע המושלם לניתוח נתונים.

במפגש הבא (אל דאגה, נפתור באגים יחד!): נתחבר עם ה-Tableau לגיליונות הללו. נלמד איך ליצור גרפים השוואתיים שמראים ביצועי יתר (Alpha) בין מניות, ואיך להפוך את המערכת שבניתם למרכז בקרה (Dashboard) פיננסי מקצועי.
